Claude Code

From zero to productive. A step-by-step guide to setting up and using Claude Code in the terminal.

Simply put: Claude Code is an AI assistant that can read, write, and run code on your computer. You talk to it in plain English, and it does the technical work.

Properly: Claude Code is an agentic coding harness — the scaffolding that gives an AI agent the ability to do things on your machine: read files, write code, run commands, connect to external tools, and execute multi-step workflows autonomously.

1

LLM

The intelligence. This is Claude — the AI model that understands language and can reason about problems. When most people use Claude, it's a conversation: you ask, it answers.

2

AI Agent

What happens when you give the LLM access to tools and the ability to execute. It can read files, run commands, call APIs, make decisions about what to do next, and carry out multi-step tasks on its own.

3

Harness

The system that makes it all possible. It defines what tools the agent can use, what permissions it has, what integrations it connects to, how it retains memory. Claude Code is that harness.

LLM → Agent (LLM + tools + execution) → Harness (the wiring)
step 1Open your terminal

The terminal is just a text-based way to talk to your computer. Instead of clicking folders, you type commands. On Mac, search for "Terminal" in Spotlight (Cmd + Space).

Pro tip: Ask the Claude desktop app: "How do I open and use the terminal on Mac?" It'll walk you through it.

step 2Install Claude Code
npm install -g @anthropic-ai/claude-code

This installs Claude Code globally so you can run it from anywhere.

step 3Choose how to launch
claude

Default — start here. Asks your permission before every action. You'll approve each file edit and command. Safest way to learn.

claude --permission-mode auto

The sweet spot. Runs autonomously, but a background classifier blocks dangerous operations (mass deletes, force pushes) while letting safe ones through.

claude --dangerously-skip-permissions

No guardrails. Skips all permission checks. The name is intentionally scary. Only for throwaway environments where nothing important can break.

Getting started

The default Mac terminal works, but iTerm2 is significantly better. Combined with a few shell tweaks, you get tab completion, shared history, and a cleaner experience. Let's set it all up from scratch.

step 1Open Terminal

Press Cmd + Space to open Spotlight, type Terminal, and hit Enter. This opens the built-in Mac terminal — we'll use it to install everything else.

step 2Install Homebrew

Homebrew is a package manager for Mac — it lets you install software from the terminal. Paste this command and hit Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It will ask for your password (the one you use to log into your Mac). You won't see characters as you type — that's normal. Just type it and hit Enter.

Important: When it finishes, it will show "Next steps" with two commands to run. Copy and paste those commands — they add Homebrew to your path so you can use it.

step 3Install iTerm2

Now use Homebrew to install iTerm2:

brew install --cask iterm2

Once installed, open iTerm2 from Spotlight (Cmd + Space, type iTerm). From now on, use iTerm2 instead of the default Terminal.

step 4Install Starship prompt

Starship gives you a clean, informative prompt that shows your current folder, git branch, and more:

brew install starship
step 5Configure your shell

Your terminal has a settings file called .zshrc that runs every time you open a new window. We're going to add some settings that make it much nicer to use. You don't need to understand this file — just paste the command below into iTerm2 and hit Enter.

This one command backs up your existing settings (if any) and installs everything for you:

[ -f ~/.zshrc ] && cp ~/.zshrc ~/.zshrc.backup.$(date +%s); cat > ~/.zshrc << 'ENDCONFIG'
# ── Homebrew ──
eval "$(/opt/homebrew/bin/brew shellenv)"

# ── Starship prompt ──
eval "$(starship init zsh)"

# ── Tab completion (case-insensitive) ──
autoload -Uz compinit && compinit
zstyle ':completion:*' matcher-list 'm:{a-zA-Z}={A-Za-z}'
setopt COMPLETE_IN_WORD

# ── History (shared across sessions) ──
HISTSIZE=10000
SAVEHIST=10000
setopt SHARE_HISTORY
setopt HIST_IGNORE_DUPS

# ── Aliases ──
alias ll="ls -la"
alias gs="git status"
alias gc="git commit"
alias gp="git push"

# ── PATH ──
export PATH="$HOME/.local/bin:$PATH"
ENDCONFIG
source ~/.zshrc && echo "✓ Shell configured!"

When you see ✓ Shell configured! you're done. Here's what you just got:

  • Tab completion that's case-insensitive — type cd doc + Tab and it auto-completes to Documents
  • Shared history across all terminal sessions
  • Shortcuts like gs for git status, ll for listing files

Three things:

  • If you had an existing .zshrc file, it saved a backup (named with a timestamp so nothing gets lost)
  • Wrote the new shell config to ~/.zshrc
  • Reloaded the config so it takes effect immediately
iTerm2 — with your new config

Claude Code works great out of the box, but there's one customization worth doing right away: a status bar that shows you what's happening at a glance.

Add a status bar

See your model, how much context window you've used, your git branch, and current directory. The context percentage is especially useful — when it gets high, you know it's time to start a fresh session.

Claude Opus 4.6 | [██████░░░░] 60% | main | ~/Documents/code

Paste this into iTerm2 and hit Enter — it creates the status bar script and sets it up automatically:

mkdir -p ~/.claude && cat > ~/.claude/statusline.sh << 'ENDSCRIPT'
#!/usr/bin/env bash

MODEL="Claude Opus 4.6"
CONTEXT_PCT=${CLAUDE_CONTEXT_PERCENT:-0}

# Build progress bar
BAR_LEN=10
FILLED=$(( CONTEXT_PCT * BAR_LEN / 100 ))
EMPTY=$(( BAR_LEN - FILLED ))
BAR=$(printf '%0.s█' $(seq 1 $FILLED 2>/dev/null))
BAR+=$(printf '%0.s░' $(seq 1 $EMPTY 2>/dev/null))

# Git branch
BRANCH=$(git branch --show-current 2>/dev/null)

# Current dir
DIR=${PWD/#$HOME/\~}

echo "$MODEL | [$BAR] $CONTEXT_PCT% | $BRANCH | $DIR"
ENDSCRIPT
chmod +x ~/.claude/statusline.sh && echo "✓ Status bar configured!"

When you see ✓ Status bar configured! you're done. Next time you open Claude Code, the status bar will appear at the bottom.

Three things:

  • Created the ~/.claude folder if it didn't exist yet
  • Wrote a small script that displays your model, context usage, git branch, and directory
  • Made the script executable so Claude Code can run it

This is harness engineering — configuring the environment so the agent works better for you. The status line is just the start. As you get more comfortable, you can add project-specific instructions, custom permissions, and more.

Using a skill

Wispr Flow

Wispr Flow is a voice dictation tool that works everywhere on your Mac. Instead of typing prompts to Claude Code, you just talk. It's dramatically faster for describing what you want built, explaining bugs, or giving detailed instructions. Once you try it, typing prompts feels painfully slow.

Why it matters: The better your prompts are, the better Claude Code's output is. Speaking naturally lets you give more context and detail than you'd ever bother typing.

Raycast

Raycast is a Spotlight replacement that does way more. It's free and once you set it up, you'll wonder how you worked without it. Here's what to configure:

Window Mgmt Snap windows to halves, thirds, or quarters with keyboard shortcuts. Essential for putting your terminal and browser side by side.
Clipboard History Keeps everything you've copied. Huge when you're moving code, commands, or text between Claude Code sessions and other apps.
Snippets Save frequently used prompts or commands and expand them with a short keyword. Type ";;cc" and it expands to your full Claude Code launch command.
Quicklinks Jump straight to GitHub repos, Linear projects, or Notion pages from anywhere with a couple keystrokes.
File Search Faster than Spotlight for finding project files, documents, and apps on your machine.
